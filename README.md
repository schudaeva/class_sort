# Структура проекта

### Entity
- **Car** (реализует Sortable)
  
  Класс автомобиля, имплементирует Sortable

- **Sortable**
  
  Интерфейс для всех объектов, которые можно сортировать и сохранять

### Collection
- **SortableCollection**
  
  Базовый интерфейс для всех коллекций Sortable объектов.
  
  *Назначение:*
  - Определяет единый способ работы с коллекциями объектов
  - Позволяет использовать разные реализации (список, множество и т.д.)

- **SortableArrayList** (реализует SortableCollection)
  
  Кастомная реализация динамического массива (Доп. задание 3*).
  
  *Назначение:*
  - Собственная реализация списка на основе массива
  - Динамическое расширение при заполнении

- **FileSaver**
  
  Сохранение объектов в файл (Доп. задание 2), запись объектов в файл в режиме добавления

### Fill
- **DataFiller**
  
  Базовый интерфейс для всех стратегий заполнения данных.
  
  *Назначение:*
  - Определяет единый способ получения данных из разных источников
  - Позволяет легко добавлять новые источники
  
  *Основные методы:*
  - `fill(int length)` - возвращает массив объектов указанной длины
  - `getName()` - возвращает название стратегии для меню

- **FileFiller**
  
  Чтение данных из файла.
  
  *Назначение:*
  - Читает данные из текстового файла
  - Парсит строки и создает объекты
  - Пропускает некорректные строки
  
  *Поля:*
  - `filename` - имя файла для чтения
  - `factory` - фабрика для создания объектов из строк

- **ManualFiller**
  
  Ручной ввод данных с консоли с валидацией.
  
  *Назначение:*
  - Запрашивает у пользователя данные через консоль
  - Проверяет корректность ввода и повторяет запрос при ошибке

- **RandomFiller**
  
  Генерация случайных автомобилей с использованием Stream API (Доп. задание 3).

### Parallel
- **ParallelCounter**
  
  Многопоточный подсчет количества вхождений (Доп. задание 4).

### Sort
- **SortStrategy**
  
  Интерфейс для всех стратегий сортировки
  
  *Методы:*
  - `sort(Sortable[] items)` - сортирует массив объектов на месте (изменяет порядок элементов)
  - `getName()` - возвращает название стратегии для отображения в меню

- **EvenOddSortStrategy** (реализует SortStrategy)
  
  Стратегия сортировки для дополнительного задания 1.
  
  *Особенности:*
  - Сортирует только объекты с четными значениями указанного числового поля
  - Объекты с нечетными значениями остаются на своих местах

- **SortByNumericFieldStrategy** (реализует SortStrategy)
  
  Стратегия сортировки по числовому полю (мощность, год)
  
  *Поля:*
  - `fieldName` - имя числового поля для сортировки
  - `ascending` - направление сортировки (true - по возрастанию, false - по убыванию)
  
  *Основные методы:*
  - Конструкторы с указанием поля и направления сортировки
  - `sort()` - реализация сортировки

- **SortByStringFieldStrategy** (реализует SortStrategy)
  
  Стратегия сортировки по строковому полю

### Util
- **SortableFactory**
  
  Фабрика для создания Sortable объектов из строки.
  Абстрагирует процесс создания объектов

- **CarFactory**
  
  Фабрика для создания автомобилей.
  Создает объекты Car из строк

- **Menu**
  
  Главное меню приложения
